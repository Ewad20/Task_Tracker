# Fixed: CORS Configuration for Docker Frontend

**Date:** 2026-06-17  
**Issue:** Frontend on port 3000 (Docker/Nginx) couldn't communicate with backend

---

## Problem

The Gateway CORS policy only allowed origins:
- `http://localhost:5173` (Vite dev server)
- `http://localhost:4173` (Vite preview)

But the Docker frontend runs on:
- `http://localhost:3000` (Nginx)

This would cause CORS errors when the frontend tries to call the API Gateway.

---

## Solution

Updated `src/Gateway/Program.cs` to include port 3000 in the allowed origins:

```csharp
builder.Services.AddCors(options =>
{
    options.AddPolicy("spa", policy =>
    {
        policy.WithOrigins(
                "http://localhost:5173",   // Vite dev server
                "http://localhost:4173",   // Vite preview
                "http://localhost:3000",   // Docker frontend (Nginx) ✨ NEW
                "http://127.0.0.1:5173",
                "http://127.0.0.1:4173",
                "http://127.0.0.1:3000")   // ✨ NEW
            .AllowAnyHeader()
            .AllowAnyMethod()
            .AllowCredentials();
    });
});
```

## CORS Policy Summary

### Allowed Origins

| Origin | Purpose |
|--------|---------|
| `http://localhost:5173` | Vite dev server (development) |
| `http://localhost:4173` | Vite preview (testing build) |
| `http://localhost:3000` | Docker frontend (Nginx) |
| `http://127.0.0.1:5173` | Vite dev (IP address) |
| `http://127.0.0.1:4173` | Vite preview (IP address) |
| `http://127.0.0.1:3000` | Docker frontend (IP address) |

### Policy Details

- ✅ **AllowAnyHeader**: Accepts all HTTP headers
- ✅ **AllowAnyMethod**: Accepts GET, POST, PUT, DELETE, PATCH, etc.
- ✅ **AllowCredentials**: Allows cookies and authentication headers

## Troubleshooting

### Still Getting CORS Errors?

**Check 1: Gateway is Updated**
```powershell
# Rebuild Gateway
docker-compose build gateway
docker-compose up -d gateway

# Check logs
docker-compose logs gateway | grep -i cors
```

**Check 2: Browser Cache**
```
1. Clear browser cache (Ctrl+F5)
2. Open in incognito/private window
3. Try different browser
```

**Check 3: Verify Origin**
```javascript
// In browser console
console.log(window.location.origin);
// Should be: http://localhost:3000 (or 5173, 4173)
```

**Check 4: Check Response Headers**
```powershell
# Test CORS preflight
curl -X OPTIONS http://localhost:8080/api/users \
  -H "Origin: http://localhost:3000" \
  -H "Access-Control-Request-Method: GET" \
  -v
```

Should include:
```
< Access-Control-Allow-Origin: http://localhost:3000
< Access-Control-Allow-Credentials: true
< Access-Control-Allow-Methods: GET, POST, PUT, DELETE, ...
```

## Production Considerations

### For Production Deployment

The current CORS configuration allows localhost origins only. For production:

1. **Update allowed origins** to your production domains:
   ```csharp
   policy.WithOrigins(
       "https://yourapp.com",
       "https://www.yourapp.com",
       "https://app.yourapp.com")
   ```

2. **Use environment variables**:
   ```csharp
   var allowedOrigins = builder.Configuration
       .GetSection("Cors:AllowedOrigins")
       .Get<string[]>() ?? Array.Empty<string>();
   
   policy.WithOrigins(allowedOrigins)
       .AllowAnyHeader()
       .AllowAnyMethod()
       .AllowCredentials();
   ```

3. **Add to appsettings.Production.json**:
   ```json
   {
     "Cors": {
       "AllowedOrigins": [
         "https://yourapp.com",
         "https://www.yourapp.com"
       ]
     }
   }
   ```

4. **Consider security**:
   - Don't use `AllowAnyOrigin()` in production
   - Only allow specific trusted domains
   - Use HTTPS origins only
   - Consider using `SetIsOriginAllowedToAllowWildcardSubdomains()`
